# TravelMate - 개인 여행 계획 앱

HTML, CSS, JavaScript로 제작된 실시간 여행 계획 웹 애플리케이션입니다.
실제 대중교통 API를 통합하여 **상용화 가능한** 수준의 정확한 교통 정보를 제공합니다.

## 🎯 주요 기능

### 1. 아름다운 홈페이지
- 현대적인 디자인의 랜딩 페이지
- 직관적인 네비게이션
- OpenStreetMap (Leaflet.js) 기반 지도 배경

### 2. 스마트 여행 계획 (좌측 사이드바)
- **출발지**: 현재 위치 자동 인식
- **목적지**: 주소 검색 + 지도 핀 포인트 설정
- **Enter 키**로 빠른 목적지 검색
- **출발 시간 및 여행 기간** 설정

### 3. 실시간 교통수단 정보 (좌측 사이드바)
4개 탭으로 구성된 교통수단 옵션:

#### 🚌 버스
- **실제 고속버스/시외버스 운행 정보** (TAGO API)
- 정렬 옵션: 추천순 / 최소 시간 / 최소 비용
- 터미널 정보, 출발/도착 시간, 요금 표시
- 단계별 경로 표시 (출발지 → 터미널 → 도착지)

#### 🚄 기차
- **실시간 KTX, SRT, ITX-새마을 운행 정보** (TAGO API)
- 정렬 옵션: 추천순 / 최소 시간 / 최소 비용
- 역 정보, 출발/도착 시간, 요금 표시
- 각 열차별 경유 역 정보

#### 🚗 자동차
- **실시간 경로 계산** (카카오 모빌리티 API)
- 경로 옵션:
  - 고속도로 (최단시간)
  - 고속도로 (최소비용)
  - 국도
- 통행료, 예상 시간, 거리 표시
- **고속도로 휴게소 정보를 지도에 표시** (초록색 마커)

#### 🚴 자전거
- 국토종주 경로 정보
- 거리 및 예상 소요 시간

### 4. 여행 정보 페이지
- **맛집 정보**: 목적지 주변 추천 맛집
- **관광지 정보**: 인기 관광 명소
- **지도 시각화**: 출발지-목적지 경로 표시
- 네이버 지도 스타일 (좌측 리스트 + 우측 지도)

### 5. 사용자 경험 개선
- **내 위치 버튼**: 우측 하단에서 언제든 현재 위치로 이동
- **지도가 항상 보임**: 사이드바로 정보를 표시하여 지도가 항상 보임
- **부드러운 애니메이션**: 사이드바 슬라이드, 탭 전환
- **홈페이지는 처음 한 번만**: 첫 방문 후 다시 표시하지 않음

## 🔑 API 키 설정

### 필수 설정 (실시간 교통정보 이용 시)

이 앱은 **실제 상용화를 고려하여** 여러 공공 및 상용 API를 통합했습니다.
API 키 없이도 작동하지만, 샘플 데이터만 표시됩니다.

#### 1. 공공데이터포털 API 키

[공공데이터포털](https://www.data.go.kr/)에서 다음 API 신청:

**TAGO 고속버스정보 API**
- URL: https://www.data.go.kr/data/15098522/openapi.do
- 용도: 실시간 고속버스/시외버스 운행 정보

**TAGO 열차정보 API**
- URL: https://www.data.go.kr/data/15098552/openapi.do
- 용도: KTX, SRT, ITX-새마을 운행 정보

#### 2. 카카오 모빌리티 API 키

[카카오 모빌리티 개발자 센터](https://developers.kakaomobility.com/)에서 발급:

**경로 안내 API**
- 용도: 자동차 경로 계산 (고속도로/국도)
- 통행료, 시간, 거리 정보 제공

#### 3. 한국도로공사 고속도로 휴게소 정보 API

[한국도로공사 데이터 포털](http://data.ex.co.kr/)에서 발급:

**휴게소 정보 API**
- 용도: 고속도로 경로상 휴게소 위치 표시

#### 4. ODsay API (선택사항)

[ODsay Lab](https://lab.odsay.com/)에서 발급:

**대중교통 통합 정보 API**
- 용도: 도심 내 지하철/버스 정보 (도심 내 이동)

### API 키 입력

`app.js` 파일을 열고 다음 부분에 발급받은 API 키를 입력하세요:

```javascript
// ========== API 키 설정 ==========
// ODsay API 키 (https://lab.odsay.com/ 에서 발급)
const ODSAY_API_KEY = 'YOUR_ODSAY_API_KEY';

// 공공데이터포털 API 키 (https://www.data.go.kr/ 에서 발급)
// TAGO 고속버스정보 API
const TAGO_BUS_API_KEY = 'YOUR_TAGO_BUS_API_KEY';

// TAGO 열차정보 API
const TAGO_TRAIN_API_KEY = 'YOUR_TAGO_TRAIN_API_KEY';

// 카카오 모빌리티 API 키 (https://developers.kakaomobility.com/ 에서 발급)
const KAKAO_MOBILITY_API_KEY = 'YOUR_KAKAO_MOBILITY_API_KEY';

// 한국도로공사 고속도로 휴게소 정보 API
const EXPRESSWAY_REST_AREA_API_KEY = 'YOUR_REST_AREA_API_KEY';
```

**주의**: API 키 없이도 앱은 정상 작동하며, 샘플 데이터를 사용합니다.


## 📖 사용 방법

### 1단계: 홈페이지에서 시작
1. "지금 시작하기" 또는 "여행 계획 시작하기" 버튼 클릭
2. 홈페이지가 슬라이드되고 지도가 보임 (한 번만 표시됨)

### 2단계: 여행 계획
1. 출발지는 현재 위치가 자동으로 입력됩니다
   - 브라우저에서 위치 권한을 허용해주세요
2. 목적지 입력창에 원하는 장소 입력 (예: "부산역", "해운대")
3. **Enter 키**를 눌러 지도에서 확인
4. 필요시 지도를 클릭하여 정확한 위치 설정
5. "위치 확인" 버튼으로 목적지 확정

**Tip**: 우측 하단의 📍 버튼을 클릭하면 언제든지 현재 위치로 지도를 이동할 수 있습니다.

### 3단계: 출발 정보 입력
1. 출발 시간 설정
2. 여행 기간 설정 (일 단위)
3. "교통수단 선택하기" 버튼 클릭

### 4단계: 교통수단 선택
1. 4개 탭 중 원하는 교통수단 선택 (버스/기차/자동차/자전거)
2. **정렬 옵션 선택** (버스/기차만):
   - 추천순: 균형잡힌 추천
   - 최소 시간: 가장 빠른 순서
   - 최소 비용: 가장 저렴한 순서
3. **경로 옵션 선택** (자동차만):
   - 고속도로 (최단시간): 가장 빠른 경로
   - 고속도로 (최소비용): 통행료가 적은 경로
   - 국도: 고속도로를 회피하는 경로
4. 리스트에서 구체적인 옵션 선택 (클릭하면 파란색 테두리로 표시)
5. "여행 정보 보기" 버튼 클릭

### 5단계: 여행 정보 확인
1. 맛집 또는 관광지 탭 선택
2. 좌측 리스트에서 관심 장소 확인
3. 우측 지도에서 경로 및 위치 확인

## 🛠️ 기술 스택

### 프론트엔드
- **HTML5**: 시맨틱 마크업
- **CSS3**: 플렉스박스, 반응형 디자인, 애니메이션, 글래스모피즘
- **JavaScript (ES6+)**: 비동기 처리, Fetch API, 이벤트 핸들링

### 지도 & 경로
- **Leaflet.js**: 오픈소스 지도 라이브러리
- **OpenStreetMap**: 무료 지도 타일
- **Nominatim API**: 무료 지오코딩 (주소 ↔ 좌표 변환)
- **Leaflet Routing Machine**: 경로 찾기 및 시각화

### 실시간 교통정보 API
- **TAGO 버스 API**: 고속버스/시외버스 운행 정보
- **TAGO 열차 API**: KTX, SRT, ITX 운행 정보
- **카카오 모빌리티 API**: 자동차 경로 계산
- **한국도로공사 API**: 고속도로 휴게소 정보
- **ODsay API**: 도심 내 대중교통 정보

## ⚠️ 주의사항

### 1. API 키 관리
- API 키는 절대 공개 저장소에 커밋하지 마세요
- 프로덕션 환경에서는 환경 변수나 서버 사이드에서 관리하세요
- 각 API는 사용량 제한이 있으므로 확인하세요

### 2. CORS 이슈
일부 API는 CORS 정책으로 인해 브라우저에서 직접 호출이 제한될 수 있습니다.
이 경우 프록시 서버를 구축하거나 서버 사이드에서 API를 호출해야 합니다.

### 3. 위치 권한
브라우저에서 위치 정보 접근 권한을 허용해야 현재 위치가 표시됩니다.

### 4. HTTPS 필요
위치 정보 기능은 HTTPS 환경에서만 작동합니다. (localhost는 예외)

### 5. Nominatim 사용 제한
Nominatim API는 초당 1회 요청 제한이 있습니다.
상업적 용도로 대량 사용 시 자체 서버 구축을 권장합니다.

## 🐛 오류 해결

### 지도가 표시되지 않는 경우
1. 브라우저 콘솔(F12)에서 오류 메시지 확인
2. 로컬 서버로 실행하고 있는지 확인 (파일 직접 열기 X)
3. 인터넷 연결 확인 (CDN에서 라이브러리 로드)

### 교통정보가 "샘플 데이터"로 표시되는 경우
1. app.js의 API 키가 올바르게 입력되었는지 확인
2. API 키가 유효한지 확인 (만료 여부)
3. 브라우저 콘솔에서 API 오류 확인
4. CORS 이슈인 경우 프록시 서버 구축 필요

### 위치를 찾을 수 없는 경우
1. 브라우저 위치 권한 허용 확인
2. HTTPS 또는 localhost에서 실행 중인지 확인
3. 정확한 주소나 장소명 입력 (예: "서울역", "부산 해운대")

### 경로가 표시되지 않는 경우
1. 출발지와 목적지가 모두 설정되었는지 확인
2. 인터넷 연결 확인
3. 브라우저 콘솔에서 라우팅 오류 확인

## 📊 데모 사용 예시

### 서울 → 부산 여행
- 출발지: 현재 위치 (또는 "서울역")
- 목적지: "부산역" 또는 "해운대"
- 교통수단:
  - 기차 (KTX: 2시간 40분, 59,800원)
  - 버스 (고속버스: 4시간 30분, 15,000원)
  - 자동차 (고속도로: 4시간 30분, 통행료 45,000원)

### 경기도 근교 여행
- 출발지: "서울시청"
- 목적지: "수원 화성"
- 교통수단:
  - 자동차 (국도: 1시간, 통행료 0원)
  - 기차 (ITX: 30분)
